# userMigrate - 玩家数据迁移工具

## 概述
玩家数据迁移脚本，支持迁移玩家的数据  
适用于玩家原帐号丢失问题  
**自用插件**  

插件支持迁移的数据：
- iLand 插件领地/受信任数据
- 原版 nbt 数据

## 注意
1. 玩家的新旧账户必须在ll数据库内有数据

2. 迁移mc原版数据会把 旧账户的数据和新账户的数据 交换 
    比如 a 是旧帐号 b 是新的    
    合并之后a会有b的数据，b会有a的数据  
    这一步是为了防止b号上有东西，迁移了之后东西没了玩家来闹  
    （才不是懒得合并背包数据呢 >v<  

3. 迁移领地数据时会把 旧账户的数据和新账户的数据 合并  
    比如 a 是旧帐号 b 是新的  
    a 有领地 (1, 2)  
    b 有领地 (4, 5)  
    合并之后b有 (1, 2, 4, 5) 这些领地，a帐号一个领地也没有了  

4. uwu

## 如何使用
1. 加载插件
2. 后台按要求执行命令：
  - jsdebug ll.imports("userMigrate", "mc")(旧的账户名, 新的账户名) // 迁移mc原版数据
  - jsdebug ll.imports("userMigrate", "iland")(旧的账户名, 新的账户名) // 迁移iland领地数据
3. 迁移好了之后删除这个插件

## 参考
如果你不知道怎么跟玩家说迁移流程，可以参考一下我的迁移流程：
```text
使用账户迁移时，请按以下步骤操作：

1. 提供旧账户的归属证明
    为了确认您是旧账户的持有者，请提供尽可能详细的信息，例如：游戏截图、注册信息、末影箱物品等。

2. 准备新账户
    创建一个可用于登录服务器的新 Minecraft 账户，或使用一个已有的可用账户。
    请提供该账户的 完整名称 和 主界面截图 。

3. 迁移完成
    迁移完成后，旧账户中的背包数据和领地数据将被转移至新账户，旧账户的相应数据将被销毁。
    注意：目前仅支持迁移背包与领地数据，其他数据暂不支持迁移。
```
